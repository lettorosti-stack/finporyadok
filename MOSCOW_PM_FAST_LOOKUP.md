# Оптимизация поиска прожиточного минимума Москвы — 0.7.9

Изменения:

- сохранённое значение подставляется сразу и не ждёт интернет;
- значения кэшируются по фактическому периоду: год, полугодие или квартал;
- завершённые периоды не перепроверяются автоматически;
- для текущего года автоматический повторный поиск не запускается при каждом открытии формы;
- повторная интернет-проверка выполняется вручную кнопкой «Обновить»;
- для 2026 года сначала проверяется известная прямая публикация ГАРАНТа;
- вместо десятков последовательных запросов используются один точный запрос и короткий резервный список;
- число проверяемых документов ограничено;
- сетевые тайм-ауты сокращены;
- интерфейс прекращает ожидание через 12 секунд и оставляет ручной ввод доступным.

Проверено статически:

- синтаксис app.js;
- versionCode 20;
- versionName 0.7.9;
- наличие канонических ключей year / half / quarter;
- наличие ограничений времени поиска.
